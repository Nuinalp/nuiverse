Formúlarios

Exemplos e diretrizes de uso para estilos de controle de formulário, opções de layout e componentes personalizados para criar uma grande variedade de formulários.

Visão geral

Os controles de formulário do Bootstrap expandem nossos estilos de formulário Rebootados com as classes. Use essas classes para optar por exibições personalizadas para obter uma renderização mais consistente em navegadores e dispositivos.

Certifique-se de usar um atributo de tipo apropriado em todas as entradas (por exemplo, e-mail para endereço de e-mail ou número para informações numéricas) para aproveitar os controles de entrada mais recentes, como verificação de email, seleção de número e muito mais.

Controles de formulários

Para entradas de arquivo, troque o controle .form para o arquivo .form-control.

Tamanho

Defina altura de seus inputs usando as classes globais como .size-sm ou .size-lg.

Readonly

Adicione o atributo booleano readonly em um campo de texto para evitar a modificação do valor. Os campos de somente leitura aparecem mais claros (como campos desabilitados), mas mantêm o cursor padrão.

Readonly plain text

Se você quer ter elementos <input readonly> em seu projeto como texto simples, use a classe .form-control-plaintext para remover o estilo de campo de formulário padrão e preservar a margem e preenchimento corretos..

Checkboxes e radios

Default (stacked)

Por padrão, qualquer número de checkbox(s) e radios que são irmãos imediatos serão empilhados verticalmente e apropriadamente espaçados com a classe .form-check.

Inline

Deixe checkbox(s) ou radios na mesma linha adiciona a classe .form-check-inline nos elementos que possuam a classe .form-check.

Layout

São aplicados display: block e width: 100% na maioria dos elementos do NUIverse, porém nos formulários os itens serão organizados verticamente por padrão, veja os exemplos abaixo.

Form groups

A classe .form-group é a maneira mais fácil de adicionar alguma estrutura aos formulários. Ele fornece uma classe flexível que incentiva o agrupamento adequado de rótulos, controles, texto de ajuda opcional e mensagens de validação de formulário. Por padrão, ele aplica apenas margem-inferior, mas eleva estilos adicionais em .form-inline conforme necessário. Use-o com <fieldset> s, <div> s ou quase qualquer outro elemento.
